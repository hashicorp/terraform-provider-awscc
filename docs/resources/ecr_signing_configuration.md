# awscc_ecr_signing_configuration (Resource)

The AWS::ECR::SigningConfiguration resource creates or updates the signing configuration for an Amazon ECR registry.

## Example Usage

### Configure ECR repository signing

Configure ECR repository signing with AWS Code Signing for verification

~> This example is generated by Amazon Bedrock and verified to work using Terraform.

```terraform
resource "aws_signer_signing_profile" "example" {
  platform_id = "AWSLambda-SHA384-ECDSA"
  name        = "example-signing-profile"

  tags = {
    Name        = "example-signing-profile"
    Environment = "test"
  }
}

resource "awscc_ecr_signing_configuration" "example" {
  signing_profile_parameters = [{
    signing_profile_name = aws_signer_signing_profile.example.name
  }]
  signing_enabled = true
  repository_name = "example-repository"

  tags = [
    {
      key   = "Name"
      value = "example-ecr-signing-config"
    },
    {
      key   = "Environment"
      value = "test"
    }
  ]
}
```

## Schema

### Required

- `rules` (Attributes List) Array of signing rules that define which repositories should be signed and with which signing profiles. (see [below for nested schema](#nestedatt--rules))

### Read-Only

- `id` (String) Uniquely identifies the resource.
- `registry_id` (String) 12-digit AWS account ID of the ECR registry.

<a id="nestedatt--rules"></a>
### Nested Schema for `rules`

Required:

- `signing_profile_arn` (String) AWS Signer signing profile ARN to use for matched repositories.

Optional:

- `repository_filters` (Attributes List) Optional array of repository filters. If omitted, the rule matches all repositories. If provided, must contain at least one filter. Empty arrays are not allowed. (see [below for nested schema](#nestedatt--rules--repository_filters))

<a id="nestedatt--rules--repository_filters"></a>
### Nested Schema for `rules.repository_filters`

Optional:

- `filter` (String) Repository name pattern (supports '*' wildcard).
- `filter_type` (String) Type of repository filter
